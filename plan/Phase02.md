# Phase02 – الدخول والأمان والإعدادات الأولية

## الأهداف
- شاشة Login احترافية (Rounded Card)، كلمة افتراضية 1234 **مشفّرة**.
- جلسة JWT كوكي httpOnly + مطالبة تغيير كلمة السر أول دخول.
- صفحة إعدادات: عام/بيع/أمان (هياكل فقط الآن).

## المهام
1) نموذج مستخدم واحد (admin) في DB + bcrypt hash لكلمة 1234.
2) `POST /auth/login` + `GET /auth/me` + Middleware للأمان.
3) واجهة Login + حالات الخطأ + Loading.
4) نافذة “غيّر كلمتك الافتراضية” بعد أول دخول.
5) صفحات Settings (هياكل تبويبات).

## القبول
- لا يصل أي Endpoint محمي بدون JWT.
- دخول بـ 1234 → مطالبة تغييرها → تخزين جديدة مشفّرة.
- إعدادات تُحفظ في DB/Config table.

## اختبارات
- وحدة: hash/verify.
- تكامل: Login flow + منع الوصول غير المصرّح به.
